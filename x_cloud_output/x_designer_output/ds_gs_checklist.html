    Get Started Checklist

    Use this checklist to get up and running quickly with Weebly Cloud for Designers

    You need a web browser and your login information to access Weebly Cloud. Get your login credentials from your account representative.

    1. Get Familiar with the Cloud Admin, the User Portal, and the Weebly Editor

    These are the tools you, your team members (if you have any), and your clients use to create and manage sites.

    Cloud Admin

    Many of the operations needed to manage your customers’ websites are accomplished using the Cloud Admin, including the ability to create, manage, and delete sites. You access the portal from https://your-domain/admin.

    Is this note true?

    Note: If your domain is a subdomain of weeblycloud.com, then you can also access the admin from https://weeblycloud.com/admin.

    Learn about the Cloud Admin here. [[topic tba]]

    User Portal

    Your clients access sites through the User Portal at https://your-domain/portal.

    Note: If your domain is a subdomain of weeblycloud.com, then you can also access the portal from https://weeblycloud.com/portal.

    Here, they can view site stats as well as access the Weebly Editor. You configure the level of access clients have by setting site contributor permissions.

    For example, you can completely restrict a client’s access so that they can’t access the editor at all. Or you might give limited access so that the client can view their site’s pages in the Weebly Editor, but they can’t add pages or publish. This screenshot shows the Weebly Editor with restricted permissions. There’s no Publish button and when the user clicks the Pages tab, instead of being shown the UI that lets them add pages, they’re shown a list of pages they can view.

    You can also send your clients a preview of the published site. When you click the Preview button in the editor, the site displays in a preview window with a temporary URL that you can send to clients. The URL remains valid for 30 days.

    More about the User Portal here. [[topic tba]]

    Weebly Editor

    You use the Weebly editor to design websites. It’s as easy as dragging elements from the element tray and dropping them onto a page. You access the editor from the Site Detail page in the Cloud Admin.

    Learn about the Weebly Editor here. [[topic tba]]

    2. Set Your Domain

    Log into the Cloud Admin to configure your domain. You can use either a subdomain of weeblycloud.com or you can use a custom domain. This domain is used to log into the Cloud Admin (https://your-domain/admin) and the User Portal (https://your-domain/portal).

    Learn how to set it here.

    3. Brand the Cloud Admin, the User Portal, and the Weebly Editor.

    You can add your company name and logo to the Weebly Cloud tools and to the emails sent to clients when certain events happen, like giving a client access to a site. Follow these instructions to upload a logo and set your company name. And you need to set the email used as the return address on emails sent by the Cloud Admin. Click here to set the email address.

    4. Set Other Basic Settings

    You need to configure other basic settings so that the Cloud Admin displays and behaves properly for you, including time zone, default language, and date and time formats. Check out how, here.

    5. Create a Plan

    Every site you create must be associated with a plan. Plans determine the functionality available for a site. For example, the Weebly Starter Plan doesn’t allow site search, while the other plans do.

    Instead of Weebly plans, your clients choose from plans that you create. You decide which Weebly plans you want to support, and then create your own plans to associate with them. Along with the features offered by each Weebly plan, you can set things like page limits and pricing structures. You must create at least one plan.

    More here.

    6. Create a Client

    Your clients are the people you design and create web sites for. You create and manage your clients from the Cloud Admin. You need a name and an email to set up a client. Learn how here.

    7. Determine the Level of Access a Client Has

    By default, clients have [[need to check]]

    [complete access to their site when it’s created. They can view, edit, and publish the site by logging into the User Portal. You can restrict their access by setting them up as a site contributor with limited permissions.]

    [no access to their site when it’s created. But you can give them full or restricted access by setting them up as a site contributor for a site.]

    8. Create and Publish a Site

    Once you have a client, and you’ve created at least one plan, you can create a site. From the client’s detail page, it’s a simple as clicking Add Site. You’re taken to the Weebly Editor, where you can drag and drop elements to build the site. Learn about creating sites here. Once created, you edit, preview, and publish the site.

    You Can Do Even More!

    Here’re some more checklist items you may want to do. They’re not crucial to get up and running with Weebly Cloud, but may need to be done at some point.

    Set Up a Contacts

    You can set up support, billing, and technical contacts for your company. The support contact info is displayed in the Weebly Editor, and might be helpful if your clients edit their own sites. The technical and billing contact info is shown to Weebly support so they know who to talk to about different issues. If you’re the main contact at your company, then you likely don’t need to create these two contacts. More here.

    Create Team Members

    If you have people working for you, you set them up as team members. You configure their access, so they might be able to do everything you can do, or you might limit access so that they can only access clients and their sites, or only access billing. More here.

    Create Site Templates

    If you think most of your sites may start out with a similar look and feel, you can create a template that contains everything you’d like your sites to have. When you use a template, you get a big head start in creating a site, and you can fully edit the site, even the parts created with the template. Find out more about templates here.
