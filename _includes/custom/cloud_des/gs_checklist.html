You need a web browser and your login information to access Weebly Cloud. Get your login credentials from your account representative. ## 1. Get Familiar with the Cloud Admin, the User Portal, and the Weebly Editor These are the tools you, your team members (if you have any), and your clients use to create and manage sites. ### Cloud Admin {% include custom/cloud_des/inc_cloudAdmin.md %} Learn about the Cloud Admin {%if site.project=="designer"%}[here](ds_gs_admin.html) {%elsif site.project=="cloud"%}[here](cl_gs_admin.html){%endif%} . ### User Portal {% include custom/cloud_des/inc_userPortal.md %} More about the User Portal {%if site.project=="designer"%}[here](ds_gs_client_ui.html) {%elsif site.project=="cloud"%}[here](cl_gs_client_ui.html){%endif%}. ### Weebly Editor {% include custom/cloud_des/inc_weeblyEditor.md %} Learn about the Weebly Editor {%if site.project=="designer"%}[here](ds_gs_client_ui.html) {%elsif site.project=="cloud"%}[here](cl_gs_client_ui.html){%endif%}. ## 2. Set Your Domain Log into the Cloud Admin to configure your domain. You can use either a subdomain of `weeblycloud.com` or you can use a custom domain. This domain is used to log into the Cloud Admin (`https://your-domain/admin`) and the User Portal (`https://your-domain/portal`). Learn how to set it [here](ds_gs_configure.html#set-your-domain). ## 3. Brand the Cloud Admin, the User Portal, and the Weebly Editor. You can add your company name and logo to the Weebly Cloud tools and to the emails sent to clients when certain events happen, like giving a {{site.user}} access to a site. Follow these instructions to [upload a logo and set your company name](ds_gs_configure.html#brand-your-sites-and-the-cloud-admin). And you need to set the email used as the return address on emails sent by the Cloud Admin. Click [here](ds_gs_configure.html#email-url) to set the email address. ## 4. Set Other Basic Settings You need to configure other basic settings so that the Cloud Admin displays and behaves properly for you, including time zone, default language, and date and time formats. Check out how, [here](ds_gs_configure.html#configure-basic-settings). ## 5. Create a Plan Every site you create must be associated with a plan. Plans determine the functionality available for a site. For example, the Weebly Starter Plan doesn't allow site search, while the other plans do. Instead of Weebly plans, your clients choose from plans that you create. You decide which Weebly plans you want to support, and then create your own plans to associate with them. Along with the features offered by each Weebly plan, you can set things like page limits and pricing structures. You must create at least one plan. More [here](ds_gs_plans.html). ## 6. Create a Client Your clients are the people you design and create web sites for. You create and manage your clients from the Cloud Admin. You need a name and an email to set up a client. Learn how [here](ds_gs_clients.html). ## 7. Determine the Level of Access a Client Has By default, clients have [[need to check]] [complete access to their site when it's created. They can view, edit, and publish the site by logging into the User Portal. You can restrict their access by setting them up as a [site contributor](ds_gs_access_sites.html) with limited permissions.] [no access to their site when it's created. But you can give them full or restricted access by setting them up as a [site contributor](ds_gs_access_sites.html) for a site.] ## 8. Create and Publish a Site Once you have a client, and you've created at least one plan, you can create a site. From the client's detail page, it's a simple as clicking **Add Site**. You're taken to the Weebly Editor, where you can drag and drop elements to build the site. Learn about creating sites [here](ds_gs_cr_sites). Once created, you edit, preview, and publish the site. ## You Can Do Even More! Here're some more checklist items you may want to do. They're not crucial to get up and running with Weebly Cloud, but may need to be done at some point. ### Set Up a Contacts You can set up support, billing, and technical contacts for your company. The support contact info is displayed in the Weebly Editor, and might be helpful if your clients edit their own sites. The technical and billing contact info is shown to Weebly support so they know who to talk to about different issues. If you're the main contact at your company, then you likely don't need to create these two contacts. More [here](ds_gs_configure.html#set-contact-info). ### Create Team Members If you have people working for you, you set them up as team members. You configure their access, so they might be able to do everything you can do, or you might limit access so that they can only access clients and their sites, or only access billing. More [here](ds_gs_team_members). ### Create Site Templates If you think most of your sites may start out with a similar look and feel, you can create a template that contains everything you'd like your sites to have. When you use a template, you get a big head start in creating a site, and you can fully edit the site, even the parts created with the template. Find out more about templates [here](ds_gs_cr_sites.html#create-a-site-using-a-template).
